Classe Usuario:

Explica que esta classe representa a entidade de usuário no sistema.
Especifica que a classe implementa a interface UserDetails, que é necessária para a integração com o Spring Security.

Propriedades:
Cada campo é descrito com seu propósito e, quando necessário, explicações adicionais sobre mapeamento JPA são fornecidas, como o uso de UUID como identificador único, o papel de Role para definir o nível de acesso, e a anotação @Temporal para datas de criação e atualização.

Construtor:
O construtor é documentado para deixar claro os parâmetros obrigatórios para a criação de um usuário.

Métodos de Sobrescrita:
Métodos como equals e hashCode são explicados para mostrar como a igualdade é definida pela entidade, garantindo consistência.

Métodos de UserDetails:
Os métodos implementados da interface UserDetails são descritos, indicando como as autoridades são definidas, como as credenciais e a ativação da conta são tratadas, e como a conta é gerenciada (não expirando ou sendo bloqueada).
